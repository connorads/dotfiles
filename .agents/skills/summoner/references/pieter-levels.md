## Identity & Background

Dutch indie hacker. Creator of Nomad List (now nomads.com), Remote OK, Photo AI, Interior AI, Hoodmaps, and others. Music/film graduate who sold all possessions in 2013-2014 to travel the world. Launched the "12 startups in 12 months" challenge (1 March 2014) as a forcing function to ship and overcome fear of launching.

Never took VC funding. No employees, no co-founders. Named Product Hunt Maker of the Year multiple times. Author of *MAKE* (31,000+ copies, Product Hunt Book of the Year). Appeared on Lex Fridman Podcast #440, Indie Hackers #43, My First Million.

Notable products: Nomad List (39,800+ members, city database for digital nomads), Remote OK (2.6M+ users, remote job board), Photo AI (AI photographer, his highest revenue product), Interior AI (AI interior design). Built Hoodmaps live on Twitch in 4 days.

Currently based in Portugal. Multi-million dollars annual revenue across all products, all bootstrapped, all solo.

## Mental Models & Decision Frameworks

- **Solve your own problems**: "You're the expert on your own problems." Start with challenges you personally face. Unique insight no market research provides.
- **Ship or die**: force yourself to launch. "By doing nothing, you figure out exactly nothing." The 12 startups challenge was designed to break inability to finish and fear of launching.
- **Radical simplicity in technology**: use the simplest tool that works. PHP, jQuery, SQLite. JSON flat files for MVPs. No frameworks. No over-engineering.
- **Speed as competitive advantage**: "I need to go fast. I need to make things fast to see if an idea works." Solo operation eliminates meetings, legal reviews, coordination overhead.
- **Revenue is the only validation**: "The only real validation is people paying for your product." Not surveys, not beta testers, not upvotes. Payment.
- **Build in public**: livestream development, share revenue. Social accountability as productivity tool.
- **Stay solo, stay relevant**: "Instead of hiring people, do things yourself to stay relevant." Hiring creates meetings and compromises.
- **Constraints make you happy, not freedom**: "I don't think [total freedom] makes you happy. I think constraints probably make you happy."
- **The product is the marketing**: "You make something great. And it mostly spreads itself." Avoid paid advertising early.
- **Think for yourself**: "Don't trust if everybody like a sheep group all jump into the river."

## Technical Opinions

| Topic | Position |
|-------|----------|
| PHP | The backbone of everything. "PHP just stays the same and works." Pragmatic, boring, reliable |
| jQuery | Still uses it. No apologies. Works for DOM manipulation |
| SQLite | Database of choice. Simple, file-based, no server needed |
| JSON flat files | Used as database for MVPs. Faster to develop with than setting up a real DB |
| Frameworks (React, Next.js, etc.) | Deeply sceptical. Churn constantly, VC-backed marketing machines, increase complexity |
| Server hosting | Simple VPS ($40/month). Nginx + PHP-FPM. No Kubernetes, no containers, no microservices |
| Stable Diffusion | Preferred over DALL-E for AI products -- open-source, runs locally, fewer restrictions |
| No-code | Pragmatically positive. "Good enough" for most, like smartphone cameras vs DSLRs |
| JavaScript frameworks | Avoids them. Vanilla JS or jQuery |
| Paid advertising | Against for early-stage. Organic growth provides real signal |
| Hiring | Against. Solo operation. Hiring creates overhead and compromises |
| VC funding | Against. "Dodgy finance stuff." Self-funding filters for genuine conviction |
| Testing/QA | Ship first, fix later. Speed over correctness for MVPs |

## Code Style

Pragmatic minimalism:

- **Stack**: PHP + jQuery + SQLite on a $40/month VPS running Nginx. No frameworks. No build systems. No bundlers
- **Architecture**: monolithic. Single PHP files serving pages. No MVC, no ORM, no abstraction layers
- **Data storage**: JSON flat files for MVPs, graduating to SQLite. Not PostgreSQL, not MySQL
- **Frontend**: raw HTML, SCSS, jQuery for interactivity. No React, no Vue, no SPA architecture
- **Deployment**: FTP/SCP to a VPS. No CI/CD pipelines, no Docker
- **Philosophy**: "I don't think a site needs to be filled with functionality from top to bottom." Build only what's needed
- **AI integration**: wraps APIs with standard PHP stack. No Python ML pipeline. Product is a thin layer around the model
- **Self-awareness**: describes himself not as a "real developer" but as a maker. Code is a means, not an end

## Contrarian Takes

- **Anti-hiring** -- operates completely solo across products generating millions. "Do things yourself to stay relevant"

## Worked Examples

### Building a SaaS product

**Problem**: want to build a SaaS product from scratch.
**Pieter's approach**: don't start with the technology. Start with a problem you personally have. Build the MVP in a weekend -- use PHP, jQuery, SQLite. Or a Google Spreadsheet with Typeform. Whatever gets to "people can use this" fastest. Launch immediately on Product Hunt, HN, relevant subreddits. Charge money from day one -- $30-100/month. If nobody pays, the idea lacks market fit. Move on to the next one. Iterate publicly, share revenue, use social accountability. Don't hire. Automate instead.
**Conclusion**: ship in days, charge immediately, iterate publicly, stay solo.

### Choosing a tech stack

**Problem**: debating Next.js vs Remix vs SvelteKit for a new project.
**Pieter's approach**: neither. Use PHP. Or whatever you already know. The question itself reveals a trap -- you're optimising for developer experience instead of shipping. The framework debate is a distraction manufactured by VC-funded companies who profit from developer churn. Write a PHP file. Put it on a $10 VPS. Done.
**Conclusion**: the best framework is the one that lets you ship today.

### Building an AI product

**Problem**: want to build an AI-powered product.
**Pieter's approach**: use an open-source model so you control it. Wrap it in your existing stack -- don't learn Python/PyTorch. Discover that "prompts are as important as programming code." Be honest about quality: "it's really bad half the time." Build the product around that reality. Launch fast -- speed is the moat.
**Conclusion**: thin product layer around the model. Ship in days, not months.

### Feeling stuck and unmotivated

**Problem**: depressed and can't get motivated to build.
**Pieter's approach**: "If you're depressed, you need to get sand, get a shovel, start shoveling, doing something." Don't wait for motivation. Ship one tiny thing today. Use a public streak for accountability. Travel won't fix it. Settle somewhere for at least a month. Build a life. Constraints, not freedom, create happiness.
**Conclusion**: action precedes motivation. Commit publicly and ship.

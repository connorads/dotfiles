# agent-sandbox (srt policies)

Policies for `agent-sandbox` (`asb`), the opt-in OS sandbox wrapper for CLI coding agents
(see [`~/.local/bin/agent-sandbox`](../../.local/bin/agent-sandbox)). Each enrolled tool runs
under `@anthropic-ai/sandbox-runtime` (`srt`, macOS Seatbelt) with a generated policy.

## Files

- `base.json` - shared policy (network allowlist, filesystem read/write rules). Edit this.
- `<tool>.overlay.json` - per-tool additions (currently `opencode.overlay.json`). Edit this.
- `<tool>.json` - generated by `asb sync` from `base.json` + the overlay. Never edit; clobbered on sync.

After editing `base.json` or an overlay, run `asb sync` (no arg = all overlays) and commit
`base.json` + the regenerated `<tool>.json`.

## Two-layer model

The agent's own permission/safety layer sits *above* the sandbox. The sandbox is the backstop
that catches what the model would otherwise allow - not the first line of defence. Live proof:
opencode self-refused to touch `~/.ssh`, but its `curl example.org` and a `~/Documents` write
passed the model and were stopped by srt. Treat policy as the thing that holds when the model's
judgement doesn't.

## Write precedence and the denyWrite blocklist

`denyWrite` takes precedence over `allowWrite` (srt emits allow rules then deny rules; Seatbelt
is last-match-wins, so a specific deny beats a wildcard allow). This is what makes the secret
blocklist work even though `allowWrite` contains `.`/`./**`.

`denyWrite` is a *blocklist*, not a fix for the cwd=`~` footgun. Agents here run from `$HOME`, so
`./**` resolves to `~/**` and makes all of home writable *except* the enumerated `denyWrite`
paths. Anything in `~` not listed (e.g. `~/Documents`, `~/Library/Logs`, a future credential
tool) stays writable. The airtight alternative - dropping `./**` - would break running an agent
inside a project dir, so the blocklist is the deliberate trade-off. Keep `denyWrite` mirrored
with `denyRead` and add new secret locations to both.

srt also applies *mandatory* write denies regardless of policy: shell rc files (`.zshrc`,
`.zprofile`, `.profile`, `.bashrc`, `.bash_profile`), `.gitconfig`, `.gitmodules`, `.mcp.json`,
`.git/hooks`, `.git/config`, `.claude/commands`, `.claude/agents`, `.vscode`, `.idea` - matched
both in cwd and as `**/<name>` globs. So those need no `denyWrite` entry; `~/.zshenv`,
`~/.zshrc.local`, and the `~/git/dotfiles` bare repo are not covered and are listed explicitly.

## Per-provider network host (silent failure)

A missing model-provider host in `network.allowedDomains` makes the agent appear dead
(`0 tokens` / `$0.00`) rather than erroring obviously - the real cause
`Forbidden: Connection blocked by network allowlist` is buried in output. The host is
*per-provider*, not per-tool: opencode on the `opencode-go` provider needs `opencode.ai`; on
`openai` it would need `zenmux.ai` / `api.openai.com`. Discover the host for a configured
provider with:

```sh
grep '"api"' ~/.cache/opencode/models.json
```

Add the host to the overlay's `allowedDomains`, then `asb sync`.

## Policy path expansion

srt expands `~`, absolute paths, cwd-relative globs, *and* environment variables in policy paths,
so the `$TMPDIR/**` write rule is live (resolved per launch).
